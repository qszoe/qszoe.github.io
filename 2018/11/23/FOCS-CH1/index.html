FOCS CH1 绪论

图灵模型

  1. 当今计算机建立在冯诺依曼模型基础上的。

  2. 图灵机
    阿兰 ▪ 图灵（Alan Turing）在1937年首次提出通用计算设备的设想，即所有计算都有可能在一种特殊的机器上执行。

  3. 数据处理器

    如果计算机被定义为数据处理器，则可以看成是 接受输入数据、处理数据并产生输出数据 的黑盒。

    graph LR
input( 输入数据 ) --> computer[ 单任务计算机 ]
	computer --> output( 输出数据 )

  4. 通用计算机

    • 图灵模型：用于描述通用计算机的模型，在过去的模型中添加了程序元素，用以存放要求计算机对数据进行处理的指令集合。

      • 相同程序，不同输入数据，输出结果不同
      • 相同输入数据，不同程序，输出结果不同
      • 相同输入数据和程序，希望有相同的输出

    • 可编程数据处理器模型

      graph LR
input( 输入数据 ) --> computer[ 单任务计算机 ]
computer --> program(程序)
program --> output( 输出数据 )

  5. 通用图灵机

    • 对现代计算机的首次描述
    • 只要提供了合适的程序就能做任何（可计算的）运算
    • 基于通用图灵机的计算机都是在 存储器 中储存数据

冯 ▪ 诺依曼模型

冯 ▪ 诺依曼模型的结构

1944~1945年间，冯 ▪ 诺依曼提出因程序和数据在逻辑上相同的，程序也能储存在存储器中。他要求程序必须存储在内存中，这也是与早期那种只有数据存储在存储器中的计算机结构的完全不同之处。

而计算机存储单元用来存储数据和程序，这说明了数据和程序的格式是相同的。实际上，数据和程序是以**位模式（0和1的序列）**存储在内存中。

冯 ▪ 诺依曼模型中，程序是由一组数量有限的指令组成。指令执行的顺序就是一条一条地执行，也包括那些包含了跳转任务的指令。指令的执行顺序是基于冯 ▪ 诺依曼模型的计算机的初始条件。而当今的计算机是以最高效的顺序来执行程序的。

冯 ▪ 诺依曼模型计算机分为4个子系统：存储器、算数逻辑单元、控制单元、输入/输出单元

  1. 存储器

    用以存储数据和程序的区域。

  2. 算数逻辑单元（ALU）

    进行计算和逻辑运算的区域。

  3. 控制单元

对存储器、算数逻辑单元、控制单元、输入/输出单元进行控制操作的单元

  1. 输入/输出单元
  • 输入单元：从计算机外部接收输入数据和程序
  • 输出单元：将计算机的处理结果输出到外部
  • 包含辅助存储设备，如磁盘等
